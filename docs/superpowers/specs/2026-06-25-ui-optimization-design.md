# Quiz Hub UI 优化方案最终版

日期：2026-06-25

## Design Read

Reading this as: 面向个人长期刷题和多学科扩展的中文学习工具，不是儿童教育 App，也不是商业落地页；视觉语言应是 warm professional learning console，保留一点个人网站的趣味，但主界面必须克制、清晰、耐用。

设计参数：

- `DESIGN_VARIANCE: 5`：比模板站更有性格，但不做实验性页面。
- `MOTION_INTENSITY: 3`：只做反馈和状态过渡，不做炫技动画。
- `VISUAL_DENSITY: 6`：这是做题工具，信息密度应高于落地页，但不能像后台系统那样压迫。

## 外部 Skill 结论

这次参考了两个外部 skill：

- `design-taste-frontend`：用于避免 AI 模板感、emoji 拼贴、大面积渐变、三等分卡片、装饰性动效。
- `uipro-max`：用于搜索设计系统、字体、配色、UX 和图标建议。

`uipro-max` 初次搜索把项目误判为儿童教育和定价落地页，给出了 Baloo 2、Comic Neue、claymorphism、pricing-focused landing。这不适合本项目。人工校准后，本项目应采用成人学习/效率工具方向。

最终采用：

- 字体方向：优先 `Plus Jakarta Sans` 风格的现代友好 sans；中文仍用系统中文字体栈，避免引入复杂字体加载。
- 配色方向：保留当前暖橙和墨绿双主题，参考 productivity palette 的 “teal focus + action orange”，但不改掉项目已有识别度。
- UX 重点：清晰 focus、明确 hover、持续进度反馈、移动端无水平滚动、尊重 `prefers-reduced-motion`。
- 图标原则：不再用 emoji 做主界面图标。当前项目没有 React 图标链，优先做轻量自定义 SVG 图标模块。

## 总目标

把当前网站从“能用的静态题库网页”升级为“有识别度、低摩擦、可长期扩展的学习控制台”。

优先提升：

- 首页能快速进入学科和继续练习。
- 做题页阅读更舒服，操作更明确。
- 状态反馈一眼能懂，包括已选、正确、错误、已提交、收藏、错题。
- 手机端可以顺畅做题。
- 新增学科时有统一视觉规范，不再临时塞 emoji。

本轮不改变：

- 静态站架构。
- GitHub Pages 部署方式。
- localStorage 本地保存逻辑。
- 题库数据结构的大方向。
- 原有个人风格文案。

## 视觉原则

### 1. 保留个性，但 UI 不靠 emoji

“腿毛人的练习网站”可以继续作为品牌文案存在，这部分是网站的个人记忆点。但界面符号要从 emoji 切换到统一图标系统。

具体规则：

- Logo 不再用腿 emoji，改为“题签/书签/生长线/练习轨迹”的抽象符号。
- 主题切换不再用西兰花和胡萝卜，改为纸页与叶片图标。
- 学科卡片不再用 emoji，改为学科专属 SVG。
- 工具栏按钮不使用 emoji，统一为图标加文字或图标加 tooltip。

### 2. 不做儿童教育风

避免：

- 泡泡字、卡通字体。
- 高饱和紫蓝学习色。
- 大圆角玩具感卡片。
- 过于活泼的弹跳动画。

采用：

- 清晰的中文阅读层级。
- 稳定的 8px 到 12px 圆角系统。
- 低饱和纸张背景。
- 暖橙作为行动色，墨绿作为专注色。

### 3. 工具优先，不做营销页

首页第一屏的任务不是“介绍产品”，而是让用户开始做题。

首页应更像：

- 学科入口。
- 学习进度。
- 继续练习。
- 总题量和题型概览。

而不是：

- 大 hero。
- 长价值主张。
- 定价卡。
- 社交证明。
- 营销 CTA。

## 设计系统

### 颜色

保留双主题，但重新校准角色。

橙色主题：

- 页面背景：温暖纸张色，接近 `#fffaf3`。
- 内容卡片：白色或轻微暖白。
- 主行动色：当前 `#f97316` 可保留。
- 强文本：接近黑棕，不用纯黑压迫。
- 弱文本：暖灰。
- 状态绿：用于正确、完成、进度，不与主橙抢层级。
- 状态红：只用于错误。

绿色主题：

- 页面背景：深墨绿或黑绿。
- 内容卡片：比背景略亮的墨绿。
- 主行动色：绿色。
- 次行动色：暖橙少量保留，用作跨主题的品牌记忆点。
- 边框：低对比绿色边框。
- 文本：浅灰白，不使用过暗灰字。

颜色使用纪律：

- 暖橙只做主行动和高亮，不铺满大面积页面。
- 正确、错误、收藏、提交必须有固定颜色语义。
- 同一状态在首页、侧栏、题卡、统计面板中颜色一致。

### 字体

不建议立刻引入外部字体，因为 GitHub Pages 静态站对稳定性要求更高，且中文字体加载收益有限。

推荐字体栈：

```css
font-family: "Plus Jakarta Sans", -apple-system, BlinkMacSystemFont, "Segoe UI",
  "Microsoft YaHei", "PingFang SC", sans-serif;
```

第一阶段可以不加载 `Plus Jakarta Sans`，只整理系统字体和字号。后续如果要提升英文字母和数字质感，再自托管或引入该字体。

字号方向：

- 首页主标题缩小，避免占用第一屏。
- 题干字号略提升。
- 选项行高提升。
- 统计数字使用更紧凑的数字样式。
- 标签、题型、题组文字保持小而清楚。

### 圆角与阴影

采用一致规则：

- 小按钮、标签：8px。
- 卡片、题目容器：12px。
- 弹层、抽屉：14px。
- 不使用 24px 以上的大泡泡圆角。

阴影只在 hover、浮层、移动端抽屉中使用。普通卡片优先用边框和间距表达层级。

### 动效

动效只服务四件事：

- 告诉用户“点到了”。
- 告诉用户“状态变了”。
- 帮用户跟上答案区展开。
- 让进度变化更自然。

规范：

- hover 抬升最多 1px 到 2px。
- 选项选中反馈 120ms 到 180ms。
- 答案展开 220ms 到 320ms。
- 进度变化 200ms 到 300ms。
- 支持 `prefers-reduced-motion: reduce`。

不做：

- 背景粒子。
- 复杂滚动动画。
- 大面积弹跳。
- 漂浮装饰球。
- 3D 场景。

## 图标系统

这是本轮最关键的识别度升级。

### 技术路线

当前项目是原生 HTML/CSS/JS + Vite，不建议为了图标引入 React 图标库。推荐新增一个轻量图标模块：

- `js/icons.js`

导出语义图标函数或 SVG 字符串：

```js
export const Icons = {
  home: '...',
  themePaper: '...',
  themeLeaf: '...',
  dataviz: '...',
  electronics: '...',
  machineLearning: '...',
  star: '...',
  retry: '...',
  answer: '...'
};
```

渲染层只通过语义名使用：

```js
renderIcon('machineLearning')
```

后续 `subjects/subjects.js` 的 `icon` 字段从 emoji 改成：

```js
icon: 'machineLearning'
```

### 图标风格

统一规则：

- `viewBox="0 0 24 24"`。
- 线宽 1.75 到 2。
- 圆角端点。
- 主体为线性图标。
- 可有一个小面积实心色块作为记忆点。
- 不混用多套图标风格。

### 必做图标

品牌与主题：

- `brandMark`：题签、书签和练习轨迹组合。
- `themePaper`：日间纸页。
- `themeLeaf`：夜间叶片。

学科：

- `dataviz`：坐标轴、柱形、折线。
- `electronics`：电路节点、电流箭头。
- `machineLearning`：节点网络、分类边界。

导航与操作：

- `home`
- `menu`
- `filter`
- `sets`
- `types`
- `star`
- `retry`
- `convert`
- `check`
- `answer`
- `prev`
- `next`

状态：

- `correct`
- `wrong`
- `submitted`
- `unanswered`
- `favorite`

## 首页最终方案

### 信息架构

首页改成学习控制台，不再是大欢迎页。

建议结构：

1. 顶部导航
2. 紧凑 hero
3. 继续练习区域
4. 学科卡片网格
5. 总览统计
6. 简短 footer

第一屏必须能看到至少一部分学科卡片。

### 顶部导航

保留：

- 站点名。
- 首页。
- 学科快速入口。
- 主题切换。

优化：

- Logo 替换为 `brandMark`。
- 移动端学科入口收进菜单，不直接隐藏到不可用。
- 主题按钮用图标，不显示 emoji。
- 导航高度控制在 56px 到 64px。

### Hero

Hero 缩短，保留个性但降低体积。

文案方向：

- 主标题保留“腿毛人的练习网站”或“刷题不息，腿毛不止”。
- 副标题控制在一行到两行。
- 不再使用大段解释。
- stats 不放得太抢眼。

目标不是让用户欣赏首页，而是让他立刻开始练习。

### 继续练习

如果 localStorage 有记录，首页显示“继续上次练习”。

显示内容：

- 学科名。
- 上次题组。
- 当前题号或完成进度。
- 最近正确率。
- 主按钮：继续练习。

没有记录时，显示轻量空状态：

- “选择一门学科开始今天的练习”
- 不做大插画。

### 学科卡片

每张卡片包含：

- 学科 SVG 图标。
- 学科名。
- 描述。
- 题目数。
- 题组数。
- 题型标签。
- 本地进度。
- 主操作按钮。

卡片视觉：

- 不使用所有卡片同色 emoji。
- 左侧图标区域固定尺寸。
- 进度条细线，不使用厚重彩条。
- hover 只轻微抬升。

### 首页统计

总学科、总题目、总题组保留，但从 hero 中弱化成“概览条”。

建议放在学科区上方或下方，用紧凑横向布局：

- 3 学科
- 570 题目
- 10 套卷

## 做题页最终方案

### 布局

桌面端保持三栏，但重新分配视觉重量：

- 左侧：题组、题型、筛选、题号导航。
- 中间：题干、选项、答案，是最高优先级。
- 右侧：当前进度、正确率、收藏、错题、计时。

中间区域必须最安静。侧栏和统计栏服务它，不能抢它。

### 左侧栏

优化方向：

- 减少大面积高饱和橙色。
- 分区标题清楚。
- 题号点阵状态统一。
- 筛选项用 segmented control 或小按钮组。
- 把当前筛选条件显示为可取消标签。

题号状态：

- 未答：中性边框。
- 已答：淡色填充。
- 正确：绿色小标记。
- 错误：红色小标记。
- 收藏：角标星形。
- 当前题：主色描边。

### 中间题卡

题卡应像一张干净的练习纸。

优化：

- 题型标签放在题干上方。
- 题号、题组、来源信息一行显示。
- 题干行高提升。
- 图片题给稳定边框和最大宽度。
- 不让按钮和答案区贴题干太近。

题目元信息示例：

```text
机器学习 · 练习题 2 · 单选题 · 第 12 / 200 题
```

### 选项

选项是核心交互，必须比现在更细。

状态规则：

- 默认：白底/卡片底，低对比边框。
- hover：背景略变，边框略加深。
- 已选：左侧主色状态条，边框主色。
- 正确：左侧绿色状态条，右侧正确图标。
- 错误：左侧红色状态条，右侧错误图标。
- 多选：选项前可用方形 checkbox 样式。
- 单选：选项前可用圆形 radio 样式。

不要让未选项、已选项、正确项全都靠同一种橙色表达。

### 答案区

答案区改成“解析笔记”。

结构：

- 标题：参考答案 / 解析。
- 正确答案。
- 解析正文。
- 填空关键词。
- 用户答案。

填空题关键词匹配用标签显示：

- 命中：绿色细标签。
- 未命中：中性或红色细标签。

展开方式：

- 用户点击“显示答案”或提交后展开。
- 展开时中间题卡高度自然变化。
- 移动端可滚动到答案区附近，但不能突然跳到页面底部。

### 顶部工具栏

按钮层级：

1. 主按钮：检查答案 / 下一题。
2. 次按钮：显示答案、转简答。
3. 图标按钮：收藏、重做、筛选、主题。

按钮规则：

- 重要按钮保留文字。
- 纯图标按钮必须有 `title` 和可访问标签。
- toggle 状态必须明确显示开/关。
- disabled 状态要看得出来。

### 右侧统计面板

右侧统计面板是“当前学习状态”，不是装饰。

推荐模块：

- 当前题组进度。
- 本轮正确率。
- 错题数。
- 收藏数。
- 已答题数。
- 当前计时。

视觉：

- 数字大，说明小。
- 进度条细。
- 少用大卡片嵌套。
- 绿色和红色只用于状态结果。

## 移动端方案

手机端不能照搬桌面三栏。

### 移动端结构

推荐：

- 顶部：学科名、题组、当前题号、主题按钮。
- 主体：题卡、选项、答案区。
- 底部固定操作栏：上一题、检查/下一题、收藏。
- 筛选和题号列表：底部抽屉。

### 底部栏

底部栏显示：

- 当前进度，例如 `12 / 200`。
- 主操作按钮。
- 收藏按钮。
- 题号抽屉入口。

触控要求：

- 所有点击区域至少 44px。
- 底部按钮不贴屏幕边缘。
- iOS 安全区需要留白。
- 不出现横向滚动。

### 抽屉

两个抽屉：

- 题号抽屉：查看题号点阵。
- 筛选抽屉：选择题组、题型、收藏、错题。

抽屉不需要复杂动画，滑入即可。减少移动端用户在侧栏和正文之间来回跳。

## 状态与反馈

### 空状态

需要补充：

- 没有匹配题目。
- 当前筛选下没有错题。
- 当前筛选下没有收藏。
- 本地没有继续练习记录。

空状态不使用大 emoji。用小图标、短文字、一个操作按钮。

### 错误状态

需要补充：

- 题库加载失败。
- 学科配置缺失。
- 图片加载失败。

错误提示要告诉用户下一步：

- 返回首页。
- 刷新页面。
- 检查题库文件。

### Focus 与键盘

必须保留或补强：

- `Tab` 能看到焦点。
- 上一题、下一题、检查答案可键盘操作。
- 选项可以键盘聚焦。
- 不移除 outline，除非提供 `focus-visible` 替代。

目标：打开首页就能开始学习。

改动：

目标：提升阅读和答题效率。

目标：手机端可以真正刷题。

- focus-visible 样式。

验证：

- 375px、768px、1024px、1440px 截图。
- 移动端无横向滚动。
- 触控目标足够大。
- 键盘焦点可见。

## 技术约束

本轮不引入大型前端框架。

保持：

- Vite。
- 原生 JavaScript 模块。
- 静态 HTML。
- GitHub Pages。
- localStorage 本地数据。

可以新增：

- `js/icons.js`
- 少量 CSS 设计 token。
- 必要的 DOM helper。
- Playwright 截图验证脚本，如果后续需要自动化视觉检查。

暂不新增：

- React。
- Vue。
- Tailwind。
- shadcn/ui。
- 用户登录。
- 云同步。
- 后台管理。
- 在线题库编辑器。

## 文件落点

预计主要涉及：

- `index.html`：首页结构和首页图标渲染。
- `css/app.css`：全站视觉 token、组件样式、响应式。
- `subjects/subjects.js`：学科图标字段从 emoji 改成语义名。
- `js/icons.js`：新增图标系统。
- `js/06-render.js`：做题页图标、选项、答案、状态渲染。
- `js/07-actions.js`：必要时调整交互状态触发。
- `js/11-app.js`：必要时接入移动端抽屉或初始化逻辑。

避免大范围业务重构。UI 改造应优先围绕渲染层和 CSS 进行。

## 验证标准

每阶段完成后运行：

```bash
npm run validate
npm run test:state
npm run build
npm run check:startup
```

最终 UI 验收：

- 首页主 UI 不再依赖 emoji。
- 首页第一屏能进入学科。
- 有记录时能继续练习。
- 做题页题干和选项更易读。
- 选中、正确、错误、已提交、收藏状态清晰。
- 橙色主题不刺眼。
- 绿色主题不沉闷。
- 手机端可以完成一轮答题。
- 键盘 focus 可见。
- reduced-motion 生效。
- GitHub Pages 构建后页面不空白。

## 最终判断

这次 UI 优化不追求“看起来像 AI 生成的高级网页”，而是让它像一个真正会被反复打开的学习工具。

最终方向：

少一点 emoji 和欢迎页，多一点清晰入口、稳定状态、可持续扩展的图标系统。
